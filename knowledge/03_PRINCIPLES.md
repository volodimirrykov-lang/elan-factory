# 03 — Как работаем

Правила совместной работы Володимира + ИИ. Если правило нарушено — добавь его в `07_ANTI_PATTERNS.md` с ссылкой на коммит.

## Коммуникация

1. **Все ответы на русском.** Код/комменты — можно на английском.
2. **Ссылки — всегда кликабельные markdown.** Никогда голым текстом. Путь `docs/data.js:42` → `[docs/data.js:42](docs/data.js)`. URL → `[v7padel.space](https://v7padel.space)`.
3. **Короткие ответы.** 2-3 предложения, если не запрашивали детали.
4. **Не давать первую попавшуюся идею** — если есть выбор, 2-3 варианта с pros/cons и рекомендацией.
5. **Числа — из `04_CURRENT_STATE.md`.** Не копируй в другие доки. Ссылайся.
6. **Сроки в стиле 2026, не 2022.** С ИИ всё в 10-100 раз быстрее — "1 день" а не "2 недели".

## Работа с данными

1. **Не править `docs/data.js` руками.** Build artifact. Правишь `data/constants.json` или xlsx в `_raw/` → regenerate.
2. **Не коммитить xlsx.** Приватная финансовая информация.
3. **Коммитить JSON-снапшоты.** Они — audit trail изменений, читаемы в git diff.
4. **Погружаться в данные до конца** перед тем как задавать вопросы. Сначала исчерпать все доступные источники (snapshots, constants, data.js).
5. **Проверять сверку** — сумма клиентов ≈ revenue. Если расхождение >1%, расследовать.

## Работа с сайтом

1. **Mobile-first.** Любой UI сразу адаптировать под телефон — Володимир часто смотрит с мобильного.
2. **После push — `curl` проверить.** Все страницы (index + новые) должны отдавать 200, только потом говорить "готово".
3. **Кеш-бастер при UI фиксе.** Если правил CSS/JS — bump версия в query string `?v=kN`.
4. **Единая тема.** Не добавлять `:root` в отдельных страницах. Через data-theme атрибут.

## Работа с ИИ (что не делать)

1. ❌ **Не создавать документацию "впрок"** — если Володимир не спросил. `.md` только по делу.
2. ❌ **Не добавлять feature flags, backwards compat, абстракции "на будущее"** — делай простое решение сейчас.
3. ❌ **Не оставлять TODO/FIXME** в коде. Либо делай, либо ничего.
4. ❌ **Не писать многострочные docstring/комменты.** Одна строка максимум, только если *почему* неочевидно.
5. ❌ **Не принимать destructive action без подтверждения.** `rm -rf`, `git reset --hard`, `git push --force` — только после явного OK.

## Финансовые правила

1. **Касса АГ = доход группы** (не equity injection). Входит в totalRevenue.
2. **Поступление от учредителей в RAPORLAR** = "другое направление" на сторону TR-юрлица. Тоже отдельная строка revenue группы.
3. **Related-party выделять** — ФОП UA, ELAN BEAUTY PL, ELAN COSMETICS UAE, ELAN KOZMETIK TR. На DD будут первыми вопросами.
4. **Доля 7% — нигде в UI.** Только во внутренних расчётах.
5. **CAPEX — отдельно от OPEX.** Даже если оплачен через opex-канал (строка "Инвестиции оборудование" в РАСХОДЫ xlsx).
6. **Customs (таможня) — это COGS**, не OPEX. Хотя в xlsx лежит под УСЛУГИ.

## Бизнес-правила (уникальные для ELAN)

1. **Никакой "как продать больше CMO"** — CMO даёт потолок $50M. Фокус на brand-with-manufacturing (Medik8 path).
2. **Intercos — НЕ наш ориентир.** Другая модель. Наш ориентир — L'Oréal как покупатель и их последние 10 сделок.
3. **Собственный бренд ELAN — критический путь к $1B.** Запуск май 2026 — самая важная дата года.
4. **GMP + ISO + Ecocert/Cosmos — обязательны к 2027.** Без них премиум-контракты недоступны, DD провалится.

## Git и деплой

1. **Коммит-сообщения на русском.** Первая строка — что (до 60 символов). Пустая строка. Детали.
2. **Не амендить пушнутые коммиты.**
3. **Не force-push в main.**
4. **После push — ждать Pages деплой.** Если build errored — читать log, чинить причину, не отключать.

## Memory + контекст для ИИ

1. **Этот repo — single source of truth.** Любая ИИ читает `CLAUDE.md` → понимает проект.
2. **Memory (локальная Claude-память) не переносится.** Если есть важное для другой ИИ — сразу фиксируй в `knowledge/*.md` или `CLAUDE.md`.
3. **Новый факт/правило — сразу в соответствующий `.md`.** Не надейся что "запомнится".
