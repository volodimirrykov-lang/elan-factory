# 07 — Anti-Patterns

Что уже пробовали и поняли что **не надо**. Каждый пункт — с конкретным кейсом.

## Позиционирование

### ❌ Ориентироваться на Intercos как benchmark
**Кейс**: изначально делали `ceo.html` через призму Renato Semerari (Intercos CEO).
**Почему не работает**: Intercos — pure CMO, другая модель, не покупатель. L'Oréal — вот кто покупает brand-with-manufacturing. Nicolas Hieronimus ≠ Renato Semerari.
**Правило**: смотрим на ELAN глазами **покупателя** (L'Oréal), не конкурента (Intercos). Все benchmarks из последних 10 сделок L'Oréal (Medik8, CeraVe, YTTP, Aesop, Magic, IT, Takami).

### ❌ CMO-only сценарий как main goal
**Кейс**: изначально цель была $250M как CMO.
**Проблема**: CMO-only потолок $50M. Чтобы стоить $1B нужен бренд.
**Правило**: brand-with-manufacturing — main path. CMO — только baseline (floor).

## Данные

### ❌ Ручная правка `docs/data.js`
**Кейс**: несколько раз в первой итерации правил числа прямо в data.js (например Mar otherRevenue = 91020 вместо реальных 92246).
**Проблема**: становится непонятно что источник — xlsx или мои опечатки. При следующем xlsx — теряется.
**Правило**: `data.js` — **build artifact**. Правится только через `scripts/build_data.py` из `data/constants.json` или xlsx.

### ❌ Hardcoded клиенты вместо xlsx
**Кейс**: список клиентов с суммами был в data.js вручную.
**Проблема**: каждый новый месяц нужно перепечатывать.
**Правило**: клиенты extract из RAPORLAR автоматически. Только metadata (имя/страна) в `constants.json`.

### ❌ Касса АГ "in" как otherRevenue
**Кейс**: пробовал брать `AG kassa in` напрямую как месячный "доход другого направления".
**Проблема**: в AG kassa смешаны `другое направление` + `свежие транши учредителей` (Гузенко 263k EUR в декабре — не доход, а вливание капитала).
**Правило**: `otherRevenue` = `Поступление от учредителей` из RAPORLAR (это конкретная, labeled строка). Касса AG — отдельная картина cash flow группы.

### ❌ Парсинг по фиксированным строкам
**Кейс**: первая версия парсера делала `row[5]` для банков.
**Проблема**: в Feb xlsx 105 строк, в Mar 141 — структура смещается. Fixed-row ломается.
**Правило**: **label-driven** парсинг — ищем по тексту в col A/G, state-машина для блоков (клиенты до "Дебеторская задолженность"). См. `scripts/parse_raporlar.py`.

### ❌ Строго по радиусу ±3 от "Всего:" для суммирования
**Кейс**: в kassa_ag сканировал ±3 колонки, получал двойной счёт (для Dec правый "Всего:" хватал и левую сумму).
**Правило**: **gap-stop** — скан от метки до **первой пустой ячейки-разделителя**. Устойчиво к любым структурам.

## Сайт

### ❌ "Доля 7%" в UI
**Кейс**: была в нескольких местах, Володимир попросил убрать.
**Причина**: это внутренняя цифра, пользователям сайта знать не нужно.
**Правило**: `ownerShare` в `constants.json` — только для внутренних расчётов user-specific метрик.

### ❌ Несколько тем с отдельными `:root` на страницах
**Кейс**: была попытка сделать 3 темы (Noir/Porcelain/Atelier), каждая со своим `:root` в отдельном CSS.
**Проблема**: конфликты, theme switcher ломался.
**Правило**: единый `style.css` с CSS-переменными, темы через `data-theme` атрибут в `style-themes.css`.

### ❌ Шрифты-serif для данных
**Кейс**: Instrument Serif italic везде — красиво, но нечитаемо для цифр.
**Правило**: sans-serif для всего что число/таблица. Serif italic — только для одного ключевого слова в h1 (акцент).

### ❌ Забыть про мобильный
**Кейс**: строил дашборд только для desktop.
**Правило**: каждая страница сразу mobile-first. Проверять на ширине 375-414px.

## Деплой

### ❌ Ручная проверка "работает ли" без curl
**Кейс**: коммитил → пушил → говорил "готово" → оказывалось Pages build errored, страница 404.
**Правило**: после push — `curl -sI URL` на все изменённые страницы. 200 — только тогда "готово". Мониторить через `Monitor` tool.

### ❌ Stuck Pages deployment — оставлять в покое
**Кейс**: V7 Pages застрял в "in_progress" на 2 часа, новые пуши висели с ошибкой "cancel X first".
**Правило**: если build не двигается >15 мин — проверь `gh api repos/.../pages/builds`. Если "in_progress" >1 часа — `gh api -X POST .../deployments/ID/statuses -f state=inactive` + empty commit trigger.

### ❌ Jekyll процесс ломает static HTML
**Кейс**: GitHub Pages пытается прогнать Jekyll, файлы с русскими именами не обрабатываются.
**Правило**: добавить пустой `.nojekyll` в корне (или в `docs/`). См. V7 `.nojekyll`.

## Документация

### ❌ Пишу markdown-доки "впрок"
**Кейс**: в первой итерации завода сделал pages.md, analysis.md, ещё пяток файлов — все потом удалили.
**Правило**: `.md` только когда конкретный запрос. Не плодить документы на будущее.

### ❌ Дублирование чисел между доками
**Кейс**: числа были в data.js, в README, в project_zavod.md — расходились.
**Правило**: числа ТОЛЬКО в `data/snapshots/*.json` + `data/constants.json`. Доки — через `04_CURRENT_STATE.md` (автогенерация) или ссылаются.

### ❌ Забыть правило — повторить ошибку
**Кейс**: Володимир сказал "ссылки кликабельные" — я через 2 ответа опять писал URL голым текстом.
**Правило**: сразу фиксировать правила в `03_PRINCIPLES.md`. Не надейся что "запомнится" для следующей сессии или другой ИИ.

## Коммуникация

### ❌ Давать первую попавшуюся идею
**Кейс**: Володимир сказал прямо: «мне нужны продуманные предложения, а не первые попавшиеся».
**Правило**: 2-3 варианта с pros/cons + рекомендация. Не одна идея.

### ❌ Сроки "из 2022 года"
**Кейс**: сказал "Level 1 — 2-4 недели, Level 2 — к концу 2026".
**Причина**: с ИИ всё в 10-100 раз быстрее. Реально Level 1 — 1 день, Level 2 — 1-2 недели.
**Правило**: пересмотр всех сроков под "с ИИ — быстрее". Если кажется "неделя" — скорее всего "день".

### ❌ Длинные объяснения когда спросили "да/нет"
**Кейс**: на "делать?" писал полэкрана аргументов.
**Правило**: короткий ответ. Детали по запросу.
